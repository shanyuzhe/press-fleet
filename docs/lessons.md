# 实战教训

来自真实运营的教训，每条都已转化为机制。日期为教训被确认的时间。

## 1. 冷读通过 ≠ 句子正确（2026-07）

一篇过了结构冷读、事实核查、AI 痕迹诊断的"成稿"，加上逐句语法硬门后查出 34 个 `COPY-*` 缺陷：缺主语、比较结构不对称、指代漂移、连接词无着落。结构审和句子审是两个门，谁也不能替谁签字。
→ 机制：独立的 [copyedit-pass](../skills/press-ops/references/copyedit-pass.md)，与冷读分离。

## 2. 门禁不绑定字节就会腐烂（2026-07）

"终校通过"的状态会在后续小改中悄悄失效——改一个标题、动一处图注，都没人重跑终校。
→ 机制：终校与正文 SHA-256 绑定，写进 frontmatter 和 `copyedit.md`，校验器比对三处一致；任何正文改动自动作废该门。

## 3. AI 味的中文形态和英文词表不是一回事（2026-07）

英文 AI 检测词表（delve/robust/tapestry）对中文几乎无效。真实用户反馈"太 AI 味、太攻击性"后复盘，中文的机器味主要是：同一改判句式反复出现（"比 X 更要紧的是 Y"连用四次）、反问连发、否定式小标题、对读者和同行的俯视语气。
→ 机制：[ai-pattern-pass](../skills/press-ops/references/ai-pattern-pass.md) 明确禁用英文词表和统计阈值作为中文门禁，改用六组中文症状。

## 4. 证据图不裁剪等于没放（2026-07）

官方备案表原图 2147×2001、七行小字，手机上读者直接划过。裁到"表头 + 关键一行"并在图注披露裁剪后，才真正成为证据。
→ 机制：视觉门禁要求手机宽度预览；超大官方图裁剪到相关行并在图注与清单中披露修改。

## 5. 备案日、公告日、上线日是三个日期（2026-07）

把监管备案写成"上线"是最容易犯的时效错误。雷达的三时间戳口径（`source_created_at` / `official_announced_at` / `rediscovered_at`）在成稿层同样适用。
→ 机制：[source-radar](../skills/press-ops/references/source-radar.md) 新鲜度口径 + 事实门对时间性 claim 的单独复核。

## 6. X 帖压缩自长文必然难看（2026-07）

长文压缩出来的短帖带着长文的骨架，在 X 上像新闻通稿的残段。
→ 机制：[platforms/x](../skills/press-ops/references/platforms/x.md) 规定从证据层独立改写，禁止压缩正文。

## 7. 角色红线比全能提示词有效（2026-06 起）

一个"什么都管"的提示词会在长任务里悄悄放水；把"不许做什么"拆给角色（审计员不许改稿、写手不许造事实、终校不许当二号文风师）后，违规变得可见、可指名。
→ 机制：[team.md](../skills/press-ops/references/team.md) 每个角色带红线 + 交接契约。

## 8. 论点跑步机：复述不是论证（2026-07）

一篇通过全部门禁的稿子被用户打回，主诉"AI 味太重"。复盘发现核心论点（"攻防共同训练、防守变强则攻击升级"）在开头、小节结尾、图注和收尾处共出现 5 次，措辞高度相似——每一处单看都通顺，连起来读就是原地打转。结构冷读和逐句终校都抓不到这种"跨位置复述"，因为它不是病句也不是结构缺陷。
→ 机制：[ai-pattern-pass](../skills/press-ops/references/ai-pattern-pass.md) 第 3 组新增"论点跑步机"症状：核心论点每次重现必须携带新事实、新边界或新后果；图注必须补充正文没有的信息，不许复述正文。

## 9. 发布永远留给人（设计原则）

工具默认 dry-run，`--execute` 需要当次任务的明确授权；`published` 状态需要真实 URL 或用户确认。AI 可以把包准备到 `ready`，按钮属于人。
→ 机制：发布适配器的双段式设计 + 状态机规则。
