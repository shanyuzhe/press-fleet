# Brief：示例内容包

- 文章类型：方法说明（本仓库的自我演示）
- 读者盲区：拿到 press-fleet 的人不知道一个"通过全部门禁"的内容包长什么样，也不知道各文件之间如何互相咬合。
- 一句话承诺：读完这个包的七个文件，你能说出一篇稿子从 idea 到 ready 要过哪几道门，以及哈希绑定如何防止"改一个字不重审"。
- 主钩子：这个示例包自己就通过了校验器——你可以改动正文里的任意一个字，然后运行校验器，亲眼看到终校门作废。
- 最小背景：内容包 = 一篇稿子的全部生产档案（brief / draft / sources / copyedit / review / 资产清单 / 发布记录）。
- 论证地图：文件逐个说明 → 状态机 → 哈希实验。
- 反例/边界：本示例为教学用途，正文刻意极短；真实稿件的 sources.md 通常有 5-10 个来源和逐条 claim 对照。
- 结尾任务：引导读者动手做一次"改字→校验失败"实验。
